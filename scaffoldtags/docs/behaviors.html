Overview

This page documents how the standard scaffold renderers provided with the plugin behaves. You can copy and/or extend them for your own needs. You can also just copy only the parts that you'd like to modify, as any renderers not provided will have defaulted behavior provided by the plugin's standard scaffold renderers. In general, the renderers are designed to display similar to the default rendering of the standard Grails scaffolding output, for the sake of consistency. In the future, alternate renderer sets will be made available.

Template types

The standard renderers provide for show, list, and editor templates. The editor templates are dual-purpose, as they can be used both for edit actions as well as create actions.

Supported class types

The following class types are supported by the standard renderers. When a primitive is specified, a template named using the associated wrapper class will match the primitive. For instance, byte[] will match a template using array_java.lang.Byte.

  • show - java.util.Date and java.lang.Object
  • list - java.util.Date and java.lang.Object
  • editor - java.lang.Byte[], java.lang.Boolean, java.lang.Character, java.lang.Number, java.net.URL, java.util.Calendar, java.util.Currency, java.util.Date, java.util.Locale, and java.util.TimeZone, and embedded beans

Show styles

The following styles attributes can be used with a show template.

Many-type relations

  • widget - if set to 'embedded', will display the individual instances using a show template, rather than displaying the objects with links in a list

One-type relations

  • widget - if set to 'embedded', will display the instance using a show template, rather than displaying the object using a link

Object

By default, displays the toString() output.

  • widget - if set to 'embedded', will display the embedded bean using a show template and bean introspection

List styles

The following styles attributes can be used with a list template.

Many-type relations

  • widget - if set to 'embedded', will display the individual instances using a list template, rather than displaying the objects with links in a list

One-type relations

  • widget - if set to 'embedded', will display the instance using a show template, rather than displaying the object using a link

Object

By default, displays the toString() output.

  • widget - if set to 'embedded', will display the embedded bean using a list template and bean introspection

Editor styles

The following styles attributes can be used with an editor template.

Many-to-many type relations

By default, will display the values using a UL list, and a selector to add additional values. (No mechanism for removal yet)

  • hidden - if set to true, will not display any values but will still generate values in the form.
  • widget - if set to 'list', will display a multi-select list from which the values can be selected, rather than only displaying a UL list of values.
  • readonly - if set to true, will not provide a mechanism for adding additional values but will still submit existing values.

Many-to-one type relations

By default, will display an item selector

  • hidden - if set to true, will not display the value but will generate a hidden value in the form.
  • readonly - if set to true, will not provide a mechanism for changing the value but will still submit existing values.

One-to-many type relations

By default, will display the values using a UL list, and a link by which to create new instances. (Removal through standard CRUD pages)

  • hidden - if set to true, will not display any values but will still generate values in the form.
  • widget - if set to 'embedded', will permit editing the embedded instances using an editor template
  • readonly - if set to true, will not provide a mechanism for adding additional values but will still submit existing values.

One-to-one type relations

By default, will display an item selector

  • hidden - if set to true, will not display the value but will still generate a hidden value in the form.
  • widget - if set to 'embedded', will permit editing the embedded instance using an editor template
  • readonly - if set to true, will not provide a mechanism for adding additional values but will still submit existing values.

byte[]

Displays a file browse button

boolean

Displays a g:checkbox using styles specified by the style attributes

char

Displays a 1-character textfield

Number

By default, displays a textfield.

If a constraints map is specified, and a constraint "range" or "size" is specified, then a selector of from .. to is displayed.

If no constraints map is specified but the object is of type byte, then a selector of -128 to 127 is displayed

Object

By default, displays the toString() output.

  • widget - if set to 'embedded', will permit editing the embedded instance specified via bean interfaces using an editor template

String

By default, displays a textfield.

  • constraint inList - if specified, then will display a selector of items from which to select
  • widget - if set to 'richtext', will invoke the g:richTextEditor tag with a basepath of js/fckeditor. If set to 'textarea', will display an HTML textarea element. If set to 'password', will display an INPUT of type 'password'.
  • constraint maxSize - if specified will set a maxlength attribute on a resulting HTML INPUT element (not used fo widget type richtext).
  • readonly - if set to true, will set the HTML INPUT element to readonly (not used fo widget type richtext).
  • you can also specify additional attributes to the HTML element through the style attribute map (only when an INPUT element is displayed)

URL

By default, displays a textfield.

  • constraint inList - if specified, then will display a selector of items from which to select
  • widget - If set to 'textarea', will display an HTML textarea element. If set to 'password', will display an INPUT of type 'password'.
  • constraint maxSize - if specified will set a maxlength attribute on a resulting HTML INPUT element (not used fo widget type richtext).
  • readonly - if set to true, will set the HTML INPUT element to readonly (not used fo widget type richtext).
  • you can also specify additional attributes to the HTML element through the style attribute map (only when an INPUT element is displayed)

Calendar

By default, displays using <g:datePicker>.

  • readonly - if set to true, will not provide a means to edit the date.
  • you can also specify attributes to the datePicker tag through the style attribute map

Currency

By default, displays using <g:currencySelect>.

  • you can specify attributes to the currencySelect tag through the style attribute map

Date

By default, displays using <g:datePicker>.

  • readonly - if set to true, will not provide a means to edit the date.
  • you can also specify attributes to the datePicker tag through the style attribute map

Locale

By default, displays using <g:localeSelect>.

  • you can specify attributes to the localeSelect tag through the style attribute map

TimeZone

By default, displays using <g:timeZoneSelect>.

  • you can specify attributes to the timeZoneSelect tag through the style attribute map
